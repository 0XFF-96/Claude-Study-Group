# Participation Guide

Everything you need to take part in the **Claude Certified Architect – Foundations (CCA-F)
Study Group**. Read once, then it becomes muscle memory.

## The one rule

> **Only edit files inside your own `members/<your-github-username>/` folder.**
> `PROGRESS.md` is generated by a bot — never edit it by hand. Following this rule means
> your pushes basically never conflict with anyone else's.

---

## 1. Get started (≈5 minutes)

```bash
# Clone the repo
git clone <repo-url>
cd Claude-Study-Group

# Create your folder from the template (use your lowercase GitHub username)
cp -r members/_TEMPLATE members/<your-github-username>
```

Then open your new folder and:

1. **`profile.md`** — fill in your name, GitHub handle, **target exam date**, and **status**.
   Keep the `Target exam date:` and `Status:` lines in their existing format — the bot reads them.
2. **`progress.md`** — this is your personal checklist (a copy of the
   [CCA-F syllabus](syllabus/CCA-F.md)). Leave the boxes unchecked for now.
3. **`weekly-log.md`** — replace the example entry with your first real one.

Push it:

```bash
git add members/<your-github-username>
git commit -m "Add <your-github-username> to study group"
git pull --rebase     # always pull --rebase before pushing
git push
```

Within a minute the Action adds your row to [`PROGRESS.md`](PROGRESS.md). Want to see it
immediately without waiting? Run it locally:

```bash
python scripts/update_progress.py
```

---

## 2. Weekly routine

Each week, in your own folder:

1. **Tick what you've learned** in `progress.md`: change `- [ ]` to `- [x]`. Your domain
   percentages are computed straight from these boxes.
2. **Write a check-in** at the top of `weekly-log.md` under a `## YYYY-MM-DD` header:
   hours studied, what you covered, wins, where you're stuck, next week's plan. The "stuck on"
   line is gold — that's what the group helps with.
3. **Add notes** under `notes/` as you go (one file per domain or topic — your call).
4. Commit & push:

   ```bash
   git add members/<your-github-username>
   git commit -m "Week of <date>: progress + notes"
   git pull --rebase
   git push
   ```

That's it. The bot updates the overview table for you.

---

## 3. How progress is tracked

```
your progress.md checkboxes  ──┐
your profile.md (date,status)──┼──►  scripts/update_progress.py  ──►  PROGRESS.md table
your weekly-log.md (latest date)┘            (runs in GitHub Actions)
```

- **Percentages** = checked boxes ÷ total boxes, per domain.
- **Last check-in** = the most recent `## YYYY-MM-DD` date in your `weekly-log.md`.
- **Status** = the `Status:` value in your `profile.md` (`in-progress` / `final-prep` / `passed`).

The Action runs **every Monday**, **on every push** that touches `members/`, and **on demand**
(Actions tab → "Update progress overview" → Run workflow).

### Keep the format parser-friendly

- In `progress.md`, keep the `## Domain 1` … `## Domain 5` headers exactly.
- In `profile.md`, keep `**Target exam date**: ...` and `**Status**: ...` lines.
- In `weekly-log.md`, use `## YYYY-MM-DD` headers for entries.

If your row shows `—` or wrong numbers, one of those formats drifted — check it and push again.

---

## 4. Etiquette

- **Stay in your lane.** Don't edit other people's folders or `PROGRESS.md`.
- **`git pull --rebase` before every push.** Keeps history clean and avoids merge bubbles.
- **Share generously.** Found a great video or cheat sheet? Add it to
  [`resources/README.md`](resources/README.md).
- **Ask in your weekly log.** Put blockers in "Stuck on" — others can answer in their PR/commit
  or in your group chat.

---

## 5. FAQ

**Do I edit `PROGRESS.md`?** No. Ever. It's bot-generated.

**My percentages look wrong.** You probably changed a `## Domain N` header or a box isn't a real
`- [ ]` / `- [x]`. Fix the format, run `python scripts/update_progress.py`, and push.

**I passed! 🎉** Set `Status: passed` in your `profile.md` and push. You'll show ✅ for the group.

**Can I add more checklist items?** Yes — add `- [ ]` lines under the right `## Domain N` in your
own `progress.md`. They'll count toward that domain's total.
